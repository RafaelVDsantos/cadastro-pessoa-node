SQL Injection e proteção do sistema

SQL Injection é um tipo de ataque que acontece quando alguém tenta inserir comandos SQL maliciosos nos campos de um sistema, como formulários. Se esses dados forem colocados diretamente dentro das consultas do banco de dados, o sistema pode executar comandos não autorizados, permitindo acesso ou alteração indevida das informações.

Para evitar esse problema, no projeto foram utilizadas consultas parametrizadas. Nesse método, a consulta SQL utiliza parâmetros (?), e os valores informados pelo usuário são enviados separadamente. Assim, o banco de dados trata esses dados apenas como informações, e não como comandos SQL.

Exemplo utilizado no sistema:

let sql = "INSERT INTO pessoa (nome, telefone, email) VALUES (?, ?, ?)";
await conn.query(sql, [nome, telefone, email]);
